# Writing Argus docs

These guides set how we write the documentation in this repo. Read them before
writing or editing anything under `docs/`, and check a draft against them before
it ships. They apply to people and to AI agents equally.

- **[Voice and tone](./voice-and-tone.md)** — how Argus docs should sound, and the
  patterns to cut.
- **[Technical writing](./technical-writing.md)** — how to structure pages, format
  code and tables, and keep terminology consistent.

## Where this comes from

Argus is a product of The Agent Deployment Company, so these docs follow ADC's
house voice, adapted here for technical documentation: clear and concise, not
long-form. The guides below are self-contained, so you don't need anything outside
this repo to write good Argus docs.

These pages are kept out of the published site (`srcExclude` in the VitePress
config). They're authoring guidance, not product documentation.
